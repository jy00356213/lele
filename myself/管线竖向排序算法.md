# 管线竖向排序算法（编高程序）

## 概要

对道路横断面中，X 轴（横向）与 Y 轴（纵向）管线交叉点进行竖向排序。
基准管线作为骨架，非基准管线在其三层空间（之上/中间/之下）中枚举定位。

---

## 专业术语

| 术语 | 说明 |
|------|------|
| **基准管线** | 用户指定的关键管线，作为竖向定位的参考骨架 |
| **非基准管线** | 需要由程序自动确定位置的管线 |
| **三层条件** | 以基准管线为参照，管线竖向分为：基准之上、基准中间、基准之下 |
| **交叉点** | 一条横向管线与一条纵向管线在平面上的交点 |

**注：平行管线互不交叉，各自独立判断。**

---

## Step 1 — 确定初始参数

| 参数 | 来源 |
|------|------|
| 地面标高 | 用户赋值 |
| 管线种类、管径规模、基准/非基准管线 | 用户赋值 |
| 覆土要求、结构壁厚等 | 自动读取数据库 |
| 横向管线根数（X） | 程序自动计算 |
| 纵向管线根数（Y） | 程序自动计算 |
| 总交叉点数量 | 程序自动计算 |

### 生成基准管线信息表

- **X 轴表格**（N 行，N = X 轴基准管线数）：
  `管线种类 | 管径规模 | 管内底 | 管外顶 | 管外底 | 覆土`

- **Y 轴表格**（N 行，N = Y 轴基准管线数）：
  `管线种类 | 管径规模 | 管内底 | 管外顶 | 管外底 | 覆土`

### 定义三层条件

以 X 轴上某基准管线为例，定义竖向三层空间：

```
  ┌──────────────┐
  │  基准之上     │ ← 确定覆土有多少
  │  (+1 分)      │
  ├──────────────┤
  │  基准中间     │ ← 确定中间空间有多少
  │  (-2 分)      │
  ├──────────────┤
  │  基准之下     │ ← 防止管线过深
  │  (-5 分)      │
  └──────────────┘
```

> **打分权重说明**：
> - 之上（+1）：正常情况，覆土合理
> - 中间（-2）：需要"挤一挤"，空间紧凑
> - 之下（-5）：管线过深，造价/施工代价大
>
> 不对称权重确保算法优先避免管线过深。

---

## Step 2 — 确定处理顺序（X 先还是 Y 先）

### 2.1 尝试 X 轴优先

1. 基于 **X 基准管线** 的三层条件
2. 判断 **Y 非基准管线** 能否满足三层条件
3. 每根 Y 非基准管线枚举 3 种可能性（之上/中间/之下），分别打分
4. 求和：**所有 Y 非基准管线的总得分**

### 2.2 尝试 Y 轴优先

方法同上，角色互换：
- 基于 **Y 基准管线** 的三层条件
- 判断 **X 非基准管线**
- 求和：**所有 X 非基准管线的总得分**

### 2.3 比较定序

```
if X轴总分 ≥ Y轴总分:
    先处理 X 轴（由 X 基准 → 确定 Y 非基准）
else:
    先处理 Y 轴（由 Y 基准 → 确定 X 非基准）
```

分数越高，说明该顺序下更多管线能在合理范围内排布。

---

## Step 3 — 按确定顺序执行（两轮接力）

### 举例：X 分数高 → 先运算 X

#### Pass 1：X 基准管线 → 确定 Y 非基准管线

```
输入：X 基准管线（三层条件）
目标：Y 非基准管线

方法：枚举法
每根 Y 非基准管线在 3 种位置（之上/中间/之下）中枚举，
选出得分最高的位置。
```

> **设计原则**：
> - 基准管线与基准管线永远不会矛盾 —— 因为初始基准管线由用户输入，若有矛盾即用户输入错误，程序应给出具体提醒（如："管线 A 与管线 B 在交叉点 C 处净距不足"）
> - 平行管线之间互不干扰，各自独立判断

#### Pass 2：Y 基准管线 + Y 非基准管线 → 确定 X 非基准管线

```
输入：Y 基准管线 + Pass 1 已确定的 Y 非基准管线
目标：X 非基准管线

方法：枚举法
```

**关键逻辑**：Pass 1 确定的 Y 非基准管线作为新增参数，与 Y 基准管线共同构成更密集的层位约束，使 Pass 2 的 X 非基准管线定位更精确。信息逐轮增值。

### 输出：管线排布清单

枚举所有管线（基准 + 非基准）的最终位置。

---

## Step 4 — 输出

按照规定的格式输出管线竖向排布结果。

---

## 算法流程图（文本版）

```
Step 1: 初始参数
    │
    ▼
Step 2: 定序
    │
    ├── 尝试 X 优先 → 计算总分
    ├── 尝试 Y 优先 → 计算总分
    │
    ▼ 取高分者
    │
Step 3: 两轮接力
    │
    ├── Pass 1: X基准 → Y非基准（枚举法）
    │       │
    │       ▼
    └── Pass 2: Y基准 + Y非基准 → X非基准（枚举法）
            │
            ▼
Step 4: 输出
```

## 工程校验建议

在实际编码中，建议增加以下校验：

| 校验项 | 触发条件 | 提示内容 |
|--------|---------|---------|
| 基准管线交叉净距 | 输入阶段 | "管线 A 与管线 B 在交叉点 C 处垂直净距不足，请调整标高" |
| 覆土厚度 | 枚举阶段 | "管线置于基准之上时覆土不满足规范要求" |
| 管线过深预警 | 枚举阶段 | "管线置于基准之下时埋深超过 ___ m，请确认" |
